MySQL Schema, Data Types, Relationships & Performance Indexes

1. Relational Database Strategy

For the 22North cost platform persistence layer, a relational database model (specifically MySQL 8) was selected. Relational schemas offer strong transactional consistency (ACID), strict foreign key constraint enforcement, and indexed lookups—features essential when tracking multiple SaaS cost environments, account budgets, and billing audit lines.

2. Entity-Relationship (ER) Diagram

The platform schema includes three core entities: accounts (storing metadata for the active customer organization), resources (containing raw inventory data rows analyzed by the insight engine), and import_runs (storing details of uploaded files).

3. Table Definitions

3.1 Table: accounts

Stores customer profiles, organizational segments, and budget thresholds.

Column Name Data Type Key Nullable Description / Constraints
id INT PK No Primary key with AUTO_INCREMENT.
name VARCHAR(120) No Organization name (e.g. Northstar SaaS).
industry VARCHAR(120) No Industry vertical.
monthly_budget DECIMAL(12,2) No Target spend limit for budget alerting.
created_at TIMESTAMP No Record insertion date (defaults to current time).

3.2 Table: resources

Maintains catalog details and metrics for individual cloud assets.

Column Name Data Type Key Nullable Description / Constraints
id INT PK No Primary key with AUTO_INCREMENT.
account_id INT FK No Foreign Key linked to accounts(id) with CASCADE DELETE.
service VARCHAR(120) No AWS service type (e.g., EC2, RDS, S3).
resource_name VARCHAR(160) No Unique name identifier for the resource.
environment VARCHAR(40) No Operational environment: prod, staging, dev, qa.
utilization_pct DECIMAL(5,2) No Mean resource utilization percentage (0.00% to 100.00%).
monthly_cost DECIMAL(12,2) No Current active spend calculated monthly.
estimated_monthly_savings DECIMAL(12,2) No Calculated cost recovery potential.
priority VARCHAR(20) No Insight priority level: high, medium, low.

3.3 Table: import_runs

Stores the file audit logs representing historical user CSV uploads.

Column Name Data Type Key Nullable Description / Constraints
id INT PK No Primary key with AUTO_INCREMENT.
file_name VARCHAR(200) No Filename of the imported CSV.
account_name VARCHAR(160) No Identified tenant name inside the import.
resource_count INT No Total resources imported in this run.
total_spend DECIMAL(12,2) No Combined monthly cost before optimization.
projected_savings DECIMAL(12,2) No Combined projected monthly savings.
mode VARCHAR(20) No Either mysql (live database) or sample.

4. Indexing & Query Optimization

To support instant analytics loading, indexes have been added to prevent table scans on fields commonly used for sorting and filtering: 

-- Index on foreign key relationship
CREATE INDEX idx_resources_account ON resources(account_id);

-- Index on service category for spend mix breakdowns
CREATE INDEX idx_resources_service ON resources(service);

-- Index on recommendation priority for filtered recommendations
CREATE INDEX idx_resources_priority ON resources(priority);

5. SQL Schema DDL (Data Definition Language)

The complete, executable schema initialization script used for MySQL database setup:

CREATE DATABASE IF NOT EXISTS cloud_cost_intelligence
CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

USE cloud_cost_intelligence;

CREATE TABLE IF NOT EXISTS accounts (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(120) NOT NULL,
  industry VARCHAR(120) NOT NULL,
  monthly_budget DECIMAL(12,2) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS resources (
  id INT AUTO_INCREMENT PRIMARY KEY,
  account_id INT NOT NULL,
  service VARCHAR(120) NOT NULL,
  resource_name VARCHAR(160) NOT NULL,
  environment VARCHAR(40) NOT NULL,
  region VARCHAR(40) NOT NULL,
  status VARCHAR(40) NOT NULL,
  utilization_pct DECIMAL(5,2) NOT NULL,
  monthly_cost DECIMAL(12,2) NOT NULL,
  owner_team VARCHAR(120) NOT NULL,
  last_used_days INT NOT NULL,
  recommendation_type VARCHAR(40) NOT NULL,
  recommendation_title VARCHAR(180) NOT NULL,
  recommendation_detail TEXT NOT NULL,
  estimated_monthly_savings DECIMAL(12,2) NOT NULL,
  priority VARCHAR(20) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT fk_resources_account FOREIGN KEY (account_id) REFERENCES accounts(id) ON DELETE CASCADE
);
