REST API Documentation

Back-End Endpoints, Data Schemas & Cost Intelligence Ingestion Contracts

1. REST API Architecture Overview

The 22North back-end is constructed using a Node.js and Express.js REST API server. It exposes endpoints designed to service the React-based frontend dashboard, process CSV binary uploads, perform rule-based utilization metrics analyses in real time, and audit cost reduction histories.

Base URL: http://localhost:4000 (in local development)

Data Format: All API requests and responses utilize standard application/json payloads, with the exception of CSV uploads which can be supplied as a raw multi-part file body or a structured JSON array.

2. API Endpoints Catalog

HTTP Method Endpoint Path Summary / Action Data Persistence Mode
GET /api/health Checks server status & database connectivity. None
GET /api/dashboard Calculates optimization cards & breakdown charts. Active (MySQL or Seed Fallback)
GET /api/resources Fetches raw resource data rows. Read-only
GET /api/meta Retrieves static customer journeys & schemas. None
GET /api/imports Retrieves logs of previous file imports. Read-only
POST /api/import Parses a new billing dataset and returns immediate analytics. Creates Import Run log
GET /api/health

Checks back-end health status and reveals if the persistence layer is actively querying MySQL or falling back to local memory datasets.

}

POST /api/import

Receives parsed rows from a billing file upload, feeds them to the cost heuristics processor, creates a persistent audit trail in import_runs, and returns a tailored dashboard dataset.

RequestBody JSON Schema:
Field Path Data Type Necessity Description
fileName string Required Name of the uploaded CSV (e.g. usage.csv).
account.monthlyBudget number Optional Custom target budget threshold. Defaults to 15000.
resources array Required Array of raw resources extracted from the file.
RequestBody JSON Example:
{
  "fileName": "june-billing.csv",
  "account": {
    "monthlyBudget": 20000
  },
  "resources": [
    {
      "service": "EC2",
      "resourceName": "staging-processor",
      "environment": "staging",
      "monthlyCost": 1200,
      "utilizationPct": 8,
      "recommendationType": "rightsize"
    }
  ]
}
Response JSON Example:
{
  "mode": "imported",
  "fileName": "june-billing.csv",
  "dashboard": { ... },
  "importRun": {
    "id": 14,
    "fileName": "june-billing.csv",
    "accountName": "june-billing",
    "resourceCount": 1,
    "totalSpend": 1200,
    "projectedSavings": 450,
    "createdAt": "2026-07-03T03:58:00.000Z"
  }
}
